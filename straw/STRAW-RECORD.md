## Response to the review — every required change, done

### 1. CRITICAL — switchbacks no longer cancel  ✅
`analyse_curvature` now splits each curving run at every **sustained sign
change** (5° hysteresis) before applying the 8° floor per single-direction
segment. Your three named windows, from the shipped `curves.jsonl`:

| window | rev 1 | rev 2 | gross turn |
|---|---|---|---|
| mi 37.9–39.0 | 2 curves, both L, 86° | **9 curves, `LRLRLRLRL`** | 508° (you measured ~560) |
| mi 22.9–23.9 | one 31° row | **3 curves, `LRL`** | 304° |
| mi 54.8–55.4 | — | **4 curves, `RLRR`** | 119° |

Every right-hand half of the serpentine is back. Overall L/R is now 71/67
(was almost entirely one direction).

### 2. REQUIRED — radius from a sliding window  ✅
Circle-fitting was wrong for this data (median ORS spacing here is ~56 m, so an
80 m window usually held one point and the Kåsa fit blew up to millions of
feet). Switched to the memo's actual primitive — **heading integration over the
window**: `R = arc_length / |Σ turn|`. It integrates several vertices, so 1.1 m
quantization can't swing it, and the round-trip radii now match to **0%**.

### 3. REQUIRED — the archive re-bakes the gameplay tables  ✅
Gameplay tables are now baked **from the decoded archive**, not the raw fetch.
So decode → re-bake reproduces them exactly (round-trip: 138≡138 curves, radii
0%, advisories identical — a hard acceptance gate that `sys.exit(1)`s on a
miss). Curve spans are kept verbatim (eps=0) plus an 80 m pad so edge windows
re-fit identically. The raw→archive cost is visible and honest:
`curves_raw_detected` 152 vs `curves` 138 — 14 borderline curves fall below the
threshold after 1.1 m quantization; that's the price of the archive, stated not
hidden.

### 4. REQUIRED — at_mi uses the leg-miles convention  ✅
`mile_scale = leg.miles / raw_ORS_miles`; every at_mi = raw cumulative × scale
(same as the grade bake). Mileage rides the raw distance mapped back through the
kept-vertex index — never re-derived from the ~1%-short decoded stream.

### 5. REQUIRED — meta header per shard  ✅
Line 1 of every shard:
```json
{"meta":{"schema":1,"data_version":"sha256:…","source":"OpenRouteService
driving-hgv (self-hosted) + OSM via Overpass (ODbL, © OpenStreetMap
contributors)","params":{"a_lat_g":0.3,"radius_window_m":80.0,…}}}
```
`data_version` is a **content hash** of the geometry payload — deterministic
(no wall-clock) and identical for a streamed vs bundled pack of the same data,
which is exactly the pin §6 of the strawman needs. All bake params ride along so
a re-bake is reproducible.

### 6. speed_limits hygiene  ✅
- Integer mph (`45`, not `45.0`).
- **Semantics (now defined):** a row at `at_mi` asserts the limit confirmed at
  that milepost and governs `[at_mi, next_row)`. Before the first row, and
  across an explicit gap, the runtime uses its terrain/city heuristic +
  `min(posted, advisory)`.
- **Gap markers.** When OSM has no matching way for more than 4 mi, the bake
  emits `{"mph": null}` at the point coverage was lost rather than carrying a
  stale value (see §"the canyon finding" below).

### 7. Connector curves — tagged, not dropped  ✅
Curves fully inside the first/last 0.75 mi carry `"connector": true`. The
straight leg's real mid-plains I-70 sweeps (mi 1.9, 4.2, 106.8) are correctly
**untagged**; only the in-town ramps are flagged. Shield-gated detection (b)
stays rejected. `apex_mi` added per your §4.1.

### 8. US-route shield matching — fixed, and the real cause found  ✅
Concurrency refs now match (`US 60;AZ 77` → US-60), corridor widened to 90 m.
But the 45→65 "gap" was **not** a matching miss: I queried the bbox and mid-
canyon US-60 (miles 10–70) has **no maxspeed-tagged ways within 500 m of the
route at all** — OSM only tags US-60 near Globe and Show Low. The `65 ref=US 70`
ways are a different highway. So the honest fix is the gap marker (#6), and
**this is the case the physics advisory layer exists for**: mid-canyon speed
guidance comes from the 133 curve advisories, not a posted limit. The canyon now
reads: `30 → 45 → null(canyon, no OSM posting) → 65 → 55 → 45 → 35`.
